The theory for the Login page is that all users that have a login come through this page and upon successful login are redirected to the appropriate landing page. Note the addition of contributor admin. Persons with this role are responsible for maintaining the contributors for a repository. This means further in the administration area they can add and remove users for their associated repositories. They can add other admins, but the initial admin for a repository has to be done by a NAFAN admin after verification.

I believe using an email for the username is appropriate because it is unique and easy for the user to remember.

An appropriate password strength can be determined and implemented. Additionally, there should be a Forgot Password link on this page.
