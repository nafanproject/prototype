NAFAN Admin

Repositories

A NAFAN Admin can perform all repository actions. These include adding repository, editing repository information, activating and deactivating a repository, and splitting and merging repositories. Unlike a Contributor Admin, a NAFAN Admin has access to all repositories in the system.

Currently the thinking is to contain the parent institute name within the repository. There has been some chatter about possibly saving that as a separate entity within the system.

Users

A NAFAN Admin can perform all user actions. These include adding users, editing user information and activating and deactivating a user. Unlike a Contributor Admin, a NAFAN Admin has access to all users in the system. A NAFAN admin also creates the initial Contributor Admin for a repository and has the capability of designating an existing user as a Contributer Admin. The repositories available to a Contributor Admin are also designated by the NAFAN Admin.

Audit

An Audit component contains information about the site usage. This could contain web statistics such as user logins, collections visited, search terms used, etc. Anything worth tracking can be reported within the Audit section.

TraceLog

The Tracelog, on the other hand, primarily is used for internal system reporting. Typically is contains any errors produced during site operation. It can also be used to log timed activities, and is often used for logging system activity during development and to track production problems.
